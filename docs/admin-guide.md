# ABLE RUNNING CHALLENGE 관리자 사용설명서

## 1. 관리자 로그인

- 접속 주소: `https://able-running-challenge.vercel.app/admin/login`
- 관리자 아이디와 비밀번호로 로그인합니다.
- 운영 계정 정보는 안전한 내부 채널에서만 관리합니다.

## 2. 관리자 메뉴

로그인 후 아래 3개 화면을 주로 사용합니다.

- `개요`
  - 전체 참가자 수
  - 지점별 참가자 수
  - 챌린지별 참가자 수
  - 승인 / 경고 / 거절 기록 수 확인
- `참가자 관리`
  - 참가자 목록 확인
  - 참가자 비활성화 / 다시 활성화
- `기록 관리`
  - 참가자 기록 검토
  - 상태 변경
  - 기록 수정
  - 최근 관리자 작업 로그 확인

## 3. 참가자 관리

### 참가자 목록 확인

- 참가자 이름, 아이디, participant_code, 지점, 챌린지, 누적 거리, 진행률을 볼 수 있습니다.

### 참가자 비활성화

- 참가자 목록에서 `비활성화` 버튼을 누르면 해당 참가자는 로그인할 수 없습니다.
- 비활성화된 참가자는 리더보드에도 반영되지 않습니다.
- 다시 사용하게 하려면 `다시 활성화` 버튼을 누릅니다.

## 4. 기록 관리

### 기록 목록 보는 방법

- 기록은 요약 상태로 접혀 있습니다.
- 각 기록을 클릭하면 상세 내용이 펼쳐집니다.
- 요약에서 바로 확인 가능한 정보
  - 참가자 이름
  - 아이디
  - participant_code
  - 날짜
  - 거리
  - 페이스
  - 현재 상태

### 상태 변경

- 펼친 뒤 `상태 변경` 영역에서 아래 중 하나를 선택합니다.
  - `승인`
  - `경고`
  - `거절`
- 필요하면 메모를 남긴 뒤 `상태 저장`을 누릅니다.

### 기록 수정

- 펼친 뒤 `기록 수정` 영역에서 날짜, 거리, 페이스, 메모를 수정할 수 있습니다.
- `관리자 수정 메모`를 남기고 `기록 수정 저장`을 누릅니다.
- 수정 후 상태는 시스템 규칙에 따라 다시 자동 판정됩니다.

## 5. 작업 로그 확인

- `기록 관리` 상단의 `작업 결과` 영역에서 최근 관리자 작업 로그를 볼 수 있습니다.
- 확인 가능한 내용
  - 작업 시각
  - 작업한 관리자 이름
  - 대상 참가자
  - 상태 변경 또는 기록 수정 여부
  - 이전 상태 -> 변경 상태
  - 관리자 메모

## 6. 상태 기준

- `승인`
  - 정상 기록
  - 누적 거리, 차트, 리더보드에 반영
- `경고`
  - 이상 기록 의심
  - 누적 거리, 차트, 리더보드에 미반영
- `거절`
  - 인정 불가 기록
  - 누적 거리, 차트, 리더보드에 미반영

## 7. 운영 시 체크 포인트

- 누적 거리와 리더보드는 `승인` 기록만 반영됩니다.
- 참가자에게는 실제로 러닝한 올바른 기록만 입력하도록 안내합니다.
- 기록이 많아질 경우 `기록 관리`에서 최근 작업 로그를 함께 보며 처리하면 편합니다.
- 테스트 참가자나 잘못 가입한 계정은 삭제 대신 `비활성화`로 관리하는 것을 권장합니다.

## 8. 운영 추천 순서

1. `개요`에서 전체 현황 확인
2. `참가자 관리`에서 신규 참가자와 비활성 상태 확인
3. `기록 관리`에서 경고 / 거절 대상 기록 검토
4. 필요한 경우 상태 변경 또는 기록 수정
5. `작업 결과` 로그로 처리 내용 확인
