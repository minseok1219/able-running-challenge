ABLE RUNNING CHALLENGE

관리자 사용설명서

참가자 관리, 기록 검토, 상태 변경, 기록 수정, 최근 작업 로그 확인까지 실제 운영 순서에 맞춰 빠르게 확인할 수 있는 관리자용 요약 문서입니다.

관리자 로그인 `/admin/login` 에서 관리자 계정으로 접속
주요 메뉴 개요 · 참가자 관리 · 기록 관리
운영 원칙 승인 기록만 누적 거리, 차트, 리더보드에 반영

1. 관리자 로그인

  • 접속 주소: https://able-running-challenge.vercel.app/admin/login
  • 운영 계정 정보는 안전한 내부 채널에서만 관리합니다.

2. 관리자 메뉴

개요

  • 전체 참가자 수 확인
  • 지점별 참가자 수 확인
  • 승인 / 경고 / 거절 기록 수 확인

참가자 관리

  • 참가자 목록과 진행 현황 확인
  • 비활성화 / 다시 활성화 처리

기록 관리

  • 참가자 기록 검토
  • 상태 변경 및 기록 수정
  • 최근 관리자 작업 로그 확인

운영 팁

  • 기록은 접기/펼치기 방식으로 검토
  • 작업 결과 패널에서 최근 수정 이력 추적

3. 참가자 관리

참가자 목록 확인

  • 이름, 아이디, participant_code, 지점, 챌린지, 누적 거리, 진행률을 확인합니다.

참가자 비활성화

  • 비활성화 시 해당 참가자는 로그인할 수 없습니다.
  • 비활성화된 참가자는 리더보드에도 반영되지 않습니다.
  • 재사용이 필요하면 다시 활성화 버튼으로 복구합니다.

4. 기록 관리

기록 검토

  • 기록은 요약 형태로 먼저 보이고, 클릭 시 상세 폼이 펼쳐집니다.
  • 요약에서 이름, 아이디, participant_code, 날짜, 거리, 페이스, 현재 상태를 확인할 수 있습니다.

상태 변경

  • 상태는 승인 / 경고 / 거절 중 하나로 변경합니다.
  • 필요하면 상태 변경 메모를 남기고 저장합니다.

기록 수정

  • 날짜, 거리, 페이스, 메모를 수정할 수 있습니다.
  • 관리자 수정 메모를 남기고 저장합니다.
  • 수정 후 상태는 시스템 규칙에 따라 다시 자동 판정됩니다.

5. 작업 로그 확인

  • 기록 관리 > 작업 결과 영역에서 최근 관리자 작업 로그를 확인합니다.
  • 확인 가능 항목: 작업 시각, 작업한 관리자, 대상 참가자, 상태 변경/기록 수정 여부, 이전 상태 → 변경 상태, 메모

6. 상태 기준

승인

정상 기록이며 누적 거리, 차트, 리더보드에 반영됩니다.

경고

이상 기록 의심 상태이며 누적 거리, 차트, 리더보드에 반영되지 않습니다.

거절

인정 불가 기록이며 누적 거리, 차트, 리더보드에 반영되지 않습니다.

운영 원칙

참가자에게는 실제로 러닝한 올바른 기록만 입력하도록 안내합니다.

7. 운영 추천 순서

  1. 개요에서 전체 현황 확인
  2. 참가자 관리에서 신규 참가자와 비활성 상태 확인
  3. 기록 관리에서 경고 / 거절 대상 검토
  4. 필요 시 상태 변경 또는 기록 수정
  5. 작업 결과 로그로 처리 내용 확인
